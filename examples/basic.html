1. Typography

Headings

HTML provides six levels of headings, from <h1> through <h6>. Above, you see h1 and h2 in action. The heading hierarchy helps structure content for better readability.

Subsection Heading (h4)

This is an h4 heading, typically used for subsections within a section. Headings use page-break-after: avoid to prevent them from being alone at the bottom of a page.

Text can be formatted in various ways: bold text for emphasis, italic text for style, and inline code for technical terms. You can also use highlighted text (if supported) to draw attention.

4. Code and Preformatted Text

Inline Code

Use html_to_pdf() to convert HTML to PDF. The function takes two parameters: an HTML string and a Config struct.

Code Blocks

use html2pdf::{html_to_pdf, Config, PaperSize};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let html = "<h1>Hello, PDF!</h1>";
    
    let config = Config::default()
        .with_paper_size(PaperSize::A4);
    
    let pdf = html_to_pdf(html, &config)?;
    std::fs::write("output.pdf", pdf)?;
    
    Ok(())
}

6. Special Boxes

ℹ️ Information

Info boxes are useful for highlighting important information, tips, or notes that readers should pay attention to. They use page-break-inside: avoid to stay together on one page.

⚠️ Warning

Warning boxes draw attention to potential issues or important caveats. Use them sparingly for maximum impact.

✓ Success

Success boxes can be used to highlight completed tasks, positive outcomes, or best practices that have been verified to work.

7. Figures and Images

Figures are self-contained content blocks that typically include an image and a caption. They use page-break-inside: avoid to ensure the image and caption stay together.

9. PrintCSS Features

Page Break Control

This document uses various page break controls throughout:

  • page-break-before: always - Forces a new page (used for cover page)
  • page-break-after: avoid - Prevents breaks after headings
  • page-break-inside: avoid - Keeps tables and boxes together
  • widows: 3 - Minimum lines at page top
  • orphans: 3 - Minimum lines at page bottom

Running Headers and Footers

This document includes a running header showing the document title and page numbers in the footer, implemented using CSS Paged Media margin boxes: 

@page {
    @top-center {
        content: "HTML2PDF Feature Showcase";
        font-size: 9pt;
    }
    @bottom-center {
        content: "Page " counter(page) " of " counter(pages);
    }
}

First Page Handling

The cover page has no header or footer, achieved with the @page :first pseudo-class: 

@page :first {
    @top-center { content: none; }
    @bottom-center { content: none; }
}
